## 5. 当前阻塞

执行 `docker compose up -d` 时，Docker 无法从 Docker Hub 拉取 MySQL 镜像：

```text
failed to resolve reference "docker.io/library/mysql:8.4"
connecting to registry-1.docker.io:443
connectex: A connection attempt failed
```

结论：Docker Engine、WSL 和 Compose 已正常可用；阻塞是 Docker Desktop 无法访问 Docker Hub 的 HTTPS 网络，并非 Compose 或 MySQL 配置错误。

建议按以下顺序处理：

1. 浏览器访问 <https://registry-1.docker.io/v2/>；出现未授权提示也代表网络可达。
2. 若浏览器也无法访问，切换可访问 Docker Hub 的网络，或使用实际可用的代理/镜像加速器。
3. 本机有代理时，在 Docker Desktop 的 `Settings -> Resources -> Proxies` 配置 HTTP/HTTPS 代理，然后执行 `Apply & Restart`。
4. 网络恢复后按“数据库”章节的命令重新拉取并启动。

PowerShell 出现 `>>` 代表上一条命令未结束或粘贴内容混入提示符。先按 `Ctrl+C`，再逐行输入命令；不要复制 `PS ...>`、`>>` 或命令输出。

## 6. 配置与安全提醒

- 根目录 `.env` 包含本地 MySQL 口令，目前是未跟踪文件。

- DeepSeek API Key、JWT 密钥尚未配置；后续只能放在后端环境变量，不能写入前端或提交到仓库。
- 前端 API 地址目前写死为本机地址，仅适用本地开发。容器化前后端时，应改为环境变量或使用 Nginx 反向代理。
